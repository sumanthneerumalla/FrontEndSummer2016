Form Structure

Form controls live inside a 'form' element and always carries the action attribute. The action attributes value is a url for the page on the server that will recieve the form information. The method attribute will specify whether the form will be sent as a get or a post . With a get method, the values in the form are added to the end of the action url. It is idea for short forms and retrieving data. The post method sends values in teh http headers. The post method should be used when users upload a file, submit sensitive data, submit a large amount of data or adds/deletes information from a database. Browsers use the get method by default.

Input tags

The input element is used to create several different form controls. The type attribute specifies which type of input form will be displayed and the name attribute defines the name of the value that is sent to the server when the data is submitted. The size attribute determines the number of character spaces to be shown and the maxlength attribute limits the number of characters a user can enter into the filled.

Text area input

The text area element is used to create multiline text input. Unline other elements this is not an empty element so it should have both an opening and closing tag. Text between the tag will apear in the box when the page loads and dissappears once the user starts typing. If the user doesn't remove the text between the tags then it will get sent along with whatever they typed. Witht the use of javascript, his text can be automatically removed once the user starts typing. Lastly, the cols and rows attributes can be used in this element to specify how large the text box should be, although the task is handled by CSS in modern sites
